Phase 1: Modular Front-End Architecture (React)

Transform static pages into interactive, component-based applications using React and ES6 best practices.

Technical Objectives

  • Understand how JSX renders through React’s virtual DOM engine.
  • Structure projects using components that separate logic, style, and data.
  • Use props and state to create dynamic, reusable UI elements.
  • Integrate Bootstrap and custom CSS for consistent styling and branding.
  • Organize folders (`/components`, `/assets`, `/data`) for scalable React apps.

LinkedIn Learning Courses

Milestone Projects

  • Project 1: Set up a React workspace using Vite or Create React App and mirror your Healthcare Playbook folder structure.
  • Project 2: Create a <PersonaCard /> component that renders persona details from a JSON file in the `/data/` directory.
  • Project 3: Build a reusable <SolutionCard /> component and display a grid using Bootstrap’s responsive columns.
  • Project 4: Deploy your first React build to Vercel as a sandbox demo for KMBS stakeholders.

Mini Code Snippet

// components/PersonaCard.jsx
import React from "react";

export default function PersonaCard({ name, title, challenges }) {
  return (
    <div className="card glass-card p-3 mb-3">
      <h5 className="text-accent">{name}</h5>
      <p><strong>Role:</strong> {title}</p>
      <p><strong>Challenges:</strong> {challenges.join(", ")}</p>
    </div>
  );
}

// Example usage in App.jsx
<PersonaCard 
  name="CIO – Higher Education"
  title="Technology Decision Maker"
  challenges={["Integration", "Security", "Budget"]}
/>

This snippet demonstrates how to translate static persona HTML cards into dynamic React components that pull data from JSON files or APIs.

KMBS Application Context

In Phase 1, you shift from static Bootstrap pages to modular React components. Each component mirrors a playbook element (Persona, Solution, Scenario) that can later connect to Supabase or other data sources. This foundation makes your vertical playbooks dynamic, searchable, and easily maintained by future developers.
